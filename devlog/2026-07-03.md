# Dev-log: claude-consensus v0.1.0 (2026-07-02/03)

Second public release of the "free school" series. Repo: github.com/Palo-Alto-AI-Research-Lab/claude-consensus (MIT). Sibling of claude-bible (same-day v0.1.0); roadmap pain #5 shipped out of order.

## Scope shipped

- docs/PROTOCOL.md: propose/counter/accept/commit over per-machine single-writer JSONL shards; risk tiers; deterministic tier-tripwire; `tick` driver (timeouts, round cap 3, leader disagree-and-commit); guards: self-accept loop, independent verify + rubber-stamp check, split-brain detection; separate quiet human-alert channel.
- docs/BUS.md: dual-rail bus (file mailbox + group chat) with dual-send enforced by a single entry point; capability addressing (`@gpu`); single-writer storage invariant; ACK discipline with watchdog SLA; full-snapshot heartbeats; 3-layer sync self-heal.
- docs/GOVERNANCE.md: leader/follower canon over receive-only sync; tier boundary enforced at 3 independent points; machine identity tagging; remote approval token pattern; scoped authorization relay format.
- reference/: consensus.py, machine_bus.py, bus_send.py, sync_monitor.py. Stdlib-only, no dependencies.

## Sanitization pass (live system -> public)

- hostnames -> HUB-1 / LAPTOP-1; machine-alias map shipped empty with a generic example
- hardcoded chat IDs / messenger rails -> env commands: BUS_RAIL2_CMD (group-chat rail), HUMAN_ALERT_CMD (deadlock channel)
- vault-specific absolute paths -> ~/machine-bus default, MACHINE_BUS_DIR override
- owner name in behavioral strings -> "the owner"; provenance keeps real names deliberately
- Russian log strings -> English; internal incident IDs dropped, incidents themselves kept
- deterministic leak-scan over the staging tree (hostnames, chat IDs, device IDs, personal paths, secret patterns, non-ASCII ranges) as a hard pre-push gate; human review of the first push

## Verification before release

- `py_compile` on all four scripts: pass.
- Live two-machine smoke test in a throwaway bus dir (MACHINE_KEY=HUB-1 / LAPTOP-1, CONSENSUS_NO_BUS=1): full cycle propose -> pending -> accept -> commit -> verify x2 reached "globally done" with the independent-verify condition satisfied.
- Tripwire test: `propose "delete old backups" --tier 0` force-bumped to tier 2 and escalated; no auto-commit path.
- Bus test: dual-send delivered on the file rail with rail 2 unconfigured (SKIP + nudge); read advanced markers; re-read empty; empty message refused (exit 2); `--help` no longer broadcasts (exit 2); forced file-rail failure with no rail 2 returned exit 3.

## Deliberately not shipped yet

ACK-watchdog module (imported best-effort), the inbox-robot wake loop around `pending`, deeper fallback rails (email / second messenger / cloud drive). Demand via issues reorders the queue.
